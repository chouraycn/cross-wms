# v1.0.94 — 技能计数修复 + AI触发提示词完善

## TL;DR

修复了两个问题：技能安装数量显示不准确（7个装4个显示）和AI技能自动匹配能力弱。前者改进了错误提示详情，后者重写了匹配算法为模糊匹配并加入了激活反馈。

## 修改文件

| 文件 | 改动 |
|------|------|
| `src/components/Skills/AddSkillDialog.tsx` | `handleImportSelected()` 从单纯计数改为 `failedSkills[]` 数组，列出具体失败技能名和原因 |
| `src/components/CrossWmsChat/TopBarChatInput.tsx` | 新增 `fuzzyMatch()` 模糊匹配；重写 `matchSkillFromInput()` 加权评分；新增 `showAutoTip` 内联提示 |

## 1. 技能安装失败详情

**旧行为**：批量导入技能失败时只显示 `${failed} 个技能导入失败`，用户不知道哪些技能出了问题。

**新行为**：显示 `导入失败：技能A、技能B（共 N 个）`，每条失败记录都带原因。

## 2. AI触发提示词匹配改进

**匹配算法**：从精确子串 `includes()` 升级为 `fuzzyMatch()` 模糊匹配
- 精确子串匹配（包含）：得分 1.0（即完全匹配）
- 前缀匹配：得分 0.9
- 单词边界匹配：得分 0.5~0.8
- 字符依次出现检测：得分 0.4

**权重调整**：
- trigger 关键词：2~3分（旧：固定2分）
- tags：1~2分（旧：固定1分）
- 名称：3~4分（旧：固定3分）
- desc 简介：1分（新增）

**激活阈值**：从 2分 提升到 3分（降低误触发率）

**激活反馈**：技能自动匹配时，输入框内显示蓝色 "⚡ 已自动激活「技能名」" 提示，2.5秒后淡出。
