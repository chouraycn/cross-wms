## TopBarChatInput 输入框常驻展开高度修复

用户反馈：AI 对话框点击后输入框"变高"，且提示文字/光标移位变形。

### 根因（src/components/CrossWmsChat/TopBarChatInput.tsx）
- `inputExpanded` 默认 false，点击 `handleInputClick` 置 true → 输入区 minHeight 32→80、内容区 28→60、justifyContent center→flex-start，触发高度跳变与光标垂直跳位。
- 占位符仅在 `!inputValue.trim() && !inputExpanded` 显示；展开后被隐藏，改由"跟随光标的提示"(`@ 引用对话文件…`) 在 caretPos 处绝对定位渲染 → 该灰字随光标持续移动，即"提示文字变形"。

### 修改
1. `const [inputExpanded] = useState(true)` —— 从一开始就是高高度，恒定不变。
2. 删除 3 处 `setInputExpanded(false/true)`（失焦折叠、点击展开、发送后折叠）。
3. 占位符条件改 `!inputValue.trim()`（空即显示），alignItems 改 flex-start（顶部对齐光标），保持静态。
4. 彻底删除"跟随光标的提示"块，消除提示文字随光标移动。

### 结论
常驻高度 + 静态占位符 = 光标与提示文字均不再因高度跳变而位移变形。仅剩浏览器原生光标闪烁/文字增长，属正常表现。
`tsc --noEmit`：本文件零错误（全量仅剩 1 个预存 `App.tsx(804)` 错误，与本次无关）。
