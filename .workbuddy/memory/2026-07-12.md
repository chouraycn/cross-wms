- **knip 配置最终正确方案（踩坑总结，已证伪早前 `knip.config.ts` 写法）**：
  1. **必须用 `knip.json`（JSON），绝不能用 `knip.config.ts`**。knip 自带 jiti 2.7.0 **无法加载 .ts 配置**——静默失败回退默认（仅 package.json bin 当入口），产出 144+ 误报且无任何报错。JSON 走独立 JSON loader 可靠。已删 knip.ts/knip.config.ts，项目根仅留 `knip.json`。
  2. **绝不在 package.json 加 `"knip"` 字段**。会触发 `Invalid input (unrecognized_keys: 0..8)` schema 错→回退默认。knip 自动发现 knip.json 或显式 `--config knip.json`。
  3. **`@types` 行数计数不可靠**（默认扫描也报 4）。用 "Unused files" 行数判断配置是否加载：`ignoreFiles:["**/*"]` 时应为 0。
  4. **`@src/*` 别名必须在 knip workspace `paths` 声明**（根 tsconfig 只有 `@/*`，server 的 `@src/*` 在 server/tsconfig.json，根 workspace 读不到→图断链→误报）。
  5. **入口必须穷举**：动态加载路由(server/routes/*.ts via lazyRouter) + 多 CLI 入口(cli/index.ts 与 cli/src/index.ts 是两个独立 bin) 都要带 `!` 列为 entry，否则整条子图断链。
