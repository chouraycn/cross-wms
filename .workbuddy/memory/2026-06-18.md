## ReAct 引擎响应延迟修复 (09:16)

### 根因分析
用户反馈 CDF auto model 下输入内容后 AI 很长时间才反馈。定位到 6 个延迟源：

1. **ReActExecutor 构造函数每次请求创建 13 个对象**（含 LongTermMemory SQLite 连接），每条消息都重新打开 DB
2. **moderate 级别任务触发 Planner LLM 调用**——在 ReAct 主循环前多一次完整 LLM 调用（无文本流给用户）
3. **simple 路径有工具调用时做 2 次 LLM 调用**——首次推理 + 工具执行 + 二次"最终响应"生成，即使首次已有充足内容
4. **无即时 SSE 反馈**——首 token 到达前用户看不到任何响应（LLM 推理可能 10-30s）
5. **longTermMemory.search 无错误兜底**——SQLite 异常直接中断主流程
6. **ReactStrategy 每次创建新 Observer/Planner 实例**

### 修复内容

**文件 1: `server/engine/reactExecutor.ts`**
- 构造函数改为懒加载：13 个对象用 getter 按需创建，`_budgetConfig` 延迟传入
- `execute()` 入口立即推送 `react_phase` SSE 事件（"AI 正在分析您的请求..."），让用户在首 token 前看到反馈
- 重置逻辑改为 `?.reset()`（仅已创建的模块才重置，避免强制初始化）
- Phase 0 规划仅对 `complex` 级别触发 LLM 规划，`moderate` 直进主循环（省 1 次 LLM 调用）
- simple 路径首次 LLM 调用已产出 >20 字符内容时，跳过第二次"最终响应" LLM 调用
- longTermMemory.search 包裹 try-catch，检索失败不阻塞主流程

**文件 2: `server/engine/executionStrategy.ts`**
- ReactStrategy 新增 static sharedObserver/sharedPlanner，懒加载复用，避免每请求重建
