## 顶栏拖拽 + 侧边栏按钮修复 (15:39-15:45)

用户反馈两个遗留 Bug：
1. 长按顶栏不能拖拽移动窗口
2. 管理侧边栏按钮点击无效

**根因分析：**
- **Bug 1**：`WindowDragBar` 使用 MUI `sx` prop 传 `WebkitAppRegion: 'drag'`，MUI 无法正确输出该非标准 vendor-prefixed CSS 属性到构建产物（验证：构建 CSS 中仅 body 的 `no-drag`，无 `drag`）
- **Bug 2**：`WindowDragBar`(zIndex 9999, height 38px) 覆盖 `SidebarToggle`(zIndex 1300, top 10px)，pointerEvents 被拖拽条截获

**修复 (`WindowDragBar.tsx`)**:
- 移除 `sx` 中的 `WebkitAppRegion`，改用 `style={{ WebkitAppRegion: 'drag' } as React.CSSProperties}`
- React `style` prop 在处理 vendor-prefixed 属性时生成内联样式，保证运行时正确应用

**修复 (`SidebarToggle.tsx`)**:
- zIndex: 1300 → 10001（高于 WindowDragBar 的 9999）
- 添加 `style={{ WebkitAppRegion: 'no-drag' } as React.CSSProperties}` 防止 WKWebView 误解

**验证**：tsc --noEmit 0 错误，vite build 成功（14324 modules）
- `SettingsPopover.tsx`：新增 `onOpenSystemAuthorization` prop，菜单点击触发 Dialog（关闭 Popover → 打开独立弹窗），移除内联渲染

## AI 对话框输入 Bug 修复 (15:34-15:37)

用户反馈两个 Bug：
1. 中文输入法输入英文有问题
2. 回车不能发送内容

**根因**：`TopBarChatInput.tsx` 中 `compositionJustEndedRef` 死锁
- `onCompositionEnd` → 设置 `compositionJustEndedRef=true`
- `handleKeyDown(Enter)` 被该标记阻塞
- 标记仅在 Enter 成功处理时清除，但 Enter 被阻塞 → 永不重置 → 所有后续回车永久失效

**修复** (`handleKeyDown` 开头):
- 保存 `const justEndedComposition = compositionJustEndedRef.current`
- 立即清除 `compositionJustEndedRef.current = false`
- Enter 判断用保存值 `!justEndedComposition`，不再依赖 ref 当前值
- 同时添加 `handleInputClick` 兜底清除

**验证**：tsc --noEmit 通过，0 错误。
