## AI 对话"生成内容时报错重试"修复（测试：给我制作个个人简历HTML）
- 根因在客户端 `src/hooks/useAgentChat.ts`，非渲染崩溃（已确认无 react-markdown/ErrorBoundary）
- 缺陷 1：`isRetryableError` 对有 status 的 HTTP 错误只在 502/503/504 时返回 true，但其余 4xx/5xx 会 fall through 到文本启发式（匹配 "network"/"connect"/"timeout"）；模型厂商返回 400/401/403 时消息常含这些词 → 被误判为可重试 → 反复重试死循环。修复：HTTP status 权威性优先，非 502/503/504 一律不可重试；无 status 时再按中英文文本（含"网络"/"连接"/"超时"）判断
- 缺陷 2：重试时 `initializeStreaming()` 把 `blockStateRef.assistantMessageIndex` 重置为 -1，但已追加的"半截"助手消息没从 messages 移除 → 新一轮 startAssistantMessage 再追加一个 → 重复气泡；且 conversationHistory 把残缺助手内容重发给服务端。修复：重试前 `setError(null)` + 过滤掉 `isStreaming` 的助手消息 + 清空 lastSeqRef + 重建流式状态
- `tsc --noEmit`：本次改动文件（NavList/useAgentChat/CenterPage）零错误；遗留错误仅 SkillsPage.tsx（用户并发编辑中）+ 两处 Chain* 组件（预存无关）
