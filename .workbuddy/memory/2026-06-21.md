## v1.5.199 WKWebView 缓存 bug 修复 (11:25)

### 问题
- 用户报告：安装的 v1.5.198 app 左上角显示 v1.5.197
- 根因：pywebview 使用本地 HTTP 服务器 (http://127.0.0.1:9988) 加载前端
- WKWebView 缓存了 v1.5.197 的 index.html + main-D5JJLYnP.js（含 `__APP_VERSION__="1.5.197"`）
- 升级到 v1.5.198 后，WKWebView 从缓存返回旧 index.html，引用旧 JS（hash D5JJLYnP 而非 hJFwfQfK）
- 缓存大小 133M，在 `~/Library/Caches/com.cdf.knowclow.desktop/WebKit/NetworkCache/`

### 修复
- `scripts/pywebview_app.py` `QuietHandler.end_headers()` 添加 no-cache headers:
  - `Cache-Control: no-cache, no-store, must-revalidate`
  - `Pragma: no-cache`
  - `Expires: 0`
- 清除 WKWebView 缓存（133M → 1M）
- 提交: `1e413edb` (fix) + `24e38e2b` (bump 1.5.199)
- 后台打包中 (task_id: ZfJaXl, --no-bump)

### 教训
- pywebview 本地 HTTP 服务器必须设置 no-cache headers，否则 WKWebView 会缓存本地资源
- 版本升级后 WKWebView 不会自动失效缓存，导致显示旧版本号/旧 UI
