## v1.5.184 - 修复流式响应后台卡死 Bug

### 问题
对话一直"思考中"，切换到其他窗口再回来后内容才显示。

### 根因
`useChat.ts` 的 `flushRender` 用 `requestAnimationFrame` 调度渲染。
WKWebView 在页面不可见（切换到其他 App）时完全暂停 rAF 回调，
导致 `scheduleRender()` 永远无法执行，切回前台后所有累积更新一次性 flush。

### 修复
- `requestAnimationFrame(flushRender)` → `window.setTimeout(flushRender, 16)`
- `cancelAnimationFrame(handle)` → `window.clearTimeout(timerHandle)`
- 变量名 `renderHandle` → `timerHandle`

### 效果
- 前台：~60fps 流畅渲染（setTimeout 16ms ≈ 60fps）
- 后台：浏览器自动节流到 ~1fps（Chrome/Safari 限制后台 timer 频率）
- 切回前台：立即补执行所有排队的渲染，不再"卡死"

### 2. ONNX 缓存 Bug 修复（QA 测试发现）✅
测试过程中 QA 发现 P0 LRU 缓存有两个 bug：

**Bug 1: 缓存键截断导致错误命中**
- 原代码: `text.slice(0, 200)` 作为长文本缓存键
- 问题: 前 200 字符相同的不同文本会错误命中同一缓存条目
- 修复: 长文本用 `createHash('sha256').update(text).digest('hex')` 作为键
- 新增: `import { createHash } from 'crypto'`

**Bug 2: FIFO 而非真 LRU**
- 原代码: 缓存命中时不移动条目位置 → 频繁访问的条目仍可能被淘汰
- 修复: 命中时 `delete(cacheKey)` + `set(cacheKey, cached)` → 移到 Map 末尾（最近使用）
