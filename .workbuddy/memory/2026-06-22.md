### 架构设计
- **Phase 0（流式优先）**: 立即用原始消息调 LLM，不等上下文压缩/记忆检索，首字延迟 <1s
- **Phase 1（后台增强）**: Promise.all 并行执行上下文压缩 + 记忆检索 + 复杂度评估，不阻塞流式
- **Phase 2（ReAct 补充）**: 仅 complexity.level === 'complex' 时启动 ReAct 补充

## error 事件路由 Bug 修复 (commit 2cc961b9)

### 根因
v9.0 重构时，`streamExecutor.ts` 的 `onSSEEvent` 回调核心事件列表遗漏了 `'error'` 类型，导致后端 error 事件走 `sendDebugSSE`（仅 LOG_DEBUG=1 时发送），前端永远收不到 → 卡在"思考中"。

### 修复（3 个文件）
- `server/engine/streamExecutor.ts` 第 167 行：核心事件列表添加 `'error'`
- `server/routes/chatService.ts` 第 361 行：handleFallback 路径同样添加 `'error'`
- `server/sse/sseTypes.ts`：新增 SSEErrorEvent 接口 + CORE_EVENT_TYPES 添加 `'error'`
