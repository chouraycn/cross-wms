# HS Code 海关商品编码查询与归类助手

## 概述

HS Code（Harmonized Commodity Description and Coding System，商品名称及编码协调制度）是由世界海关组织（WCO）制定的国际通用商品分类体系。全球超过 200 个国家和地区使用 HS Code 进行海关关税、贸易统计和监管。

---

## HS Code 结构说明

| 层级 | 位数 | 制定机构 | 说明 |
|------|------|----------|------|
| **HS 国际通用码** | 前 6 位 | WCO（世界海关组织） | 全球统一，每 5 年修订一次（最新：HS 2022） |
| **中国海关税则号** | 第 7-8 位 | 中国海关总署 | 中国本国子目 |
| **附加码** | 第 9-10 位 | 中国海关总署 | 用于更精细的监管（如暂定税率、出口退税等） |
| **CIQ 代码** | 第 11-13 位 | 检验检疫附加码 | 用于出入境检验检疫（非必需） |

> **示例**：`8471.30.10.00` → 84（机器类）→ 8471（自动数据处理设备）→ 8471.30（便携式）→ 8471.30.10（平板电脑）→ 8471.30.10.00（具体型号）

## 归类原则（HS 归类六大总规则）

1. **规则一**：归类应按税目条文及类注、章注确定
2. **规则二（a）**：不完整品/未制成品 → 按完整品归类（如已具备基本特征）
3. **规则二（b）**：混合物/组合物 → 按构成基本特征的材料归类
4. **规则三**：多个税目均可归入时 → (a) 列名具体优先 (b) 基本特征优先 (c) 从后归类
5. **规则四**：无法按上述规则归类 → 归入类似商品税目
6. **规则五**：包装容器 → (a) 特定容器与货品一同归类 (b) 普通包装材料与货品分别归类

## AI 辅助工作流

### 场景 1：用户询问某商品的 HS Code

1. 根据商品名称判断大类（参考「分类体系」）
2. 列出可能的候选 HS Code（通常 2-3 个）
3. 说明归类依据和可能的争议点
4. 建议用户通过单一窗口或海关确认

### 场景 2：用户提供 HS Code，询问税率

1. 使用 `web_fetch` 查询单一窗口或 hsbianma.com
2. 返回：最惠国税率、普通税率、增值税、消费税、出口退税
3. 注明税率时效性

### 场景 3：归类争议分析

1. 列出争议涉及的不同税目
2. 根据归类总规则逐条分析
3. 引用海关归类决定（如有）或预裁定案例
4. 给出倾向性建议，建议申请预裁定

### 场景 4：批量商品归类（WMS 场景）

1. 接收商品清单（品名 + 规格 + 用途 + 材质）
2. 逐项分析归类
3. 输出结构化结果：品名 | HS Code | 税率 | 监管条件
4. 标注不确定项，建议人工复核

## 注意事项

1. **HS Code 具有法律效力**：错误归类可能导致罚款、货物扣押，AI 归类建议仅供参考，最终以海关认定为准
2. **税率动态变化**：各国关税政策、自贸协定（如 RCEP）会影响税率，需要查询最新数据
3. **监管条件**：部分商品（如 3C 认证、食品、药品）需要额外许可证
4. **归类预裁定**：对于复杂商品，建议企业向直属海关申请归类预裁定（法律效力，有效期 3 年）
5. **HS 版本**：注意各国使用的 HS 版本（HS 2017 / HS 2022），中国 2024 年起全面执行 HS 2022 版税则
