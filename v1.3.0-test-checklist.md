## 一、技能远程市场（Remote Marketplace）

### 1.1 市场首页加载
- [ ] 打开 MarketplacePage，页面正常渲染，展示技能卡片列表
- [ ] 首次加载时触发全量同步（syncMarketplaceIndex），远程技能数据写入本地缓存
- [ ] 缓存未过期（7 天内）时不重复请求远程
- [ ] 网络异常时显示降级提示，不影响页面基础框架

### 1.2 分类筛选
- [ ] 点击分类标签（如"效率工具"、"开发辅助"），筛选结果正确
- [ ] 切换分类后列表刷新，无残留旧数据
- [ ] 空分类显示空状态占位

### 1.3 搜索功能
- [ ] 输入关键词搜索，结果按相关度排序
- [ ] 清空搜索框恢复全量展示
- [ ] 搜索无结果时显示空状态
- [ ] 搜索结果与分类筛选可叠加使用

### 1.4 排序
- [ ] 按"热门"排序，结果按安装量/评分降序
- [ ] 按"最近更新"排序，结果按更新时间降序
- [ ] 排序切换后页面刷新，无闪烁

### 1.5 技能详情页
- [ ] 从市场列表点击技能卡片进入 MarketplaceDetailPage
- [ ] 详情页正确展示：名称、描述、版本号、作者、标签、评分、截图
- [ ] 已安装技能显示"已安装"标识和当前版本号

### 1.6 安装流程
- [ ] 点击"安装"按钮，进入下载阶段（进度展示）
- [ ] 下载完成后自动触发安全审计（auditSecurity），审计报告可查看
- [ ] 审计通过后弹出权限确认对话框，列出所需权限
- [ ] 用户确认权限后执行安装（installSkill），安装成功提示
- [ ] 审计不通过时显示风险警告，提供"仍要安装"和"取消"选项
- [ ] 安装失败时显示错误信息，不留下不完整数据

### 1.7 权限确认
- [ ] 权限列表完整展示技能声明的所有权限
- [ ] 逐项展示权限等级（normal/dangerous/restricted）
- [ ] dangerous 权限高亮提示
- [ ] 用户可逐项开关，关闭某权限后安装时跳过该权限授予

### 1.8 更新检查
- [ ] 已安装技能有新版本时，详情页显示"更新"按钮
- [ ] 点击更新执行版本比对（checkUpdates），正确识别大/小版本差异
- [ ] 更新流程同安装流程（下载→审计→权限→激活）
- [ ] 更新后版本号正确刷新

### 1.9 评价展示和提交
- [ ] 详情页底部展示评价列表，按时间/评分排序
- [ ] 已安装用户可提交评价（星级 + 文字），提交后即时展示
- [ ] 未安装用户提交评价时提示"安装后方可评价"

### 1.10 离线降级
- [ ] 断网状态下打开市场页面，展示缓存数据并标注"离线模式"
- [ ] 离线模式下安装/更新操作提示"需要网络连接"
- [ ] 恢复网络后自动同步最新数据

## 二、语义匹配引擎（Semantic Matching Engine）

### 2.1 匹配 API 调用
- [ ] 调用 `POST /api/matching/match`，传入 query + mode，返回匹配结果
- [ ] keyword 模式返回关键词匹配结果，含匹配原因（reasons）
- [ ] semantic 模式返回语义相似度排序结果
- [ ] hybrid 模式返回融合排序结果
- [ ] context 模式结合上下文返回结果
- [ ] 传入非法 mode 返回 400 错误

### 2.2 高置信度自动激活
- [ ] 匹配置信度 ≥ 阈值（默认 0.85）的技能自动标记为 `auto_activate: true`
- [ ] 系统自动加载高置信度技能，无需用户手动确认
- [ ] 自动激活日志写入反馈表

### 2.3 中置信度候选展示
- [ ] 置信度在 [0.5, 0.85) 区间的技能标记为 `show_candidate: true`
- [ ] 候选技能在聊天界面侧栏/推荐区展示，用户可一键激活
- [ ] 候选列表按置信度降序排列

### 2.4 低置信度降级
- [ ] 置信度 < 0.5 的技能不展示推荐
- [ ] 低置信度结果不写入用户可见区域
- [ ] 日志记录低置信度匹配事件，供离线分析

### 2.5 匹配反馈提交
- [ ] 用户对匹配结果点击"有用/无用"按钮，调用 `POST /api/matching/feedback`
- [ ] 反馈记录写入数据库，含 skillId、score（+1/-1）、用户 ID
- [ ] 反馈学习模块根据历史反馈调整后续 hybrid 融合权重
- [ ] 反馈历史可查询（`GET /api/matching/feedback/:skillId`）

### 2.6 匹配引擎配置读写
- [ ] `GET /api/matching/config` 返回当前运行时配置
- [ ] `PUT /api/matching/config` 可更新配置项（semanticWeight, keywordWeight, threshold 等）
- [ ] `POST /api/matching/config/reset` 恢复默认配置
- [ ] 配置更新后立即生效，无需重启

### 2.7 嵌入生成和状态查询
- [ ] `POST /api/matching/embeddings/generate` 触发全量嵌入重建
- [ ] 生成过程中状态可查（进度/完成数/失败数）
- [ ] 增量更新：技能内容变化时仅重新生成对应嵌入
- [ ] 嵌入维度固定为 384 维，格式为 Float32Array 序列化 BLOB

### 2.8 聊天输入集成语义匹配
- [ ] 在 TopBarChatInput 输入时，自动触发语义匹配（debounce 300ms）
- [ ] 匹配结果在 SkillMatchResult 组件中展示
- [ ] 用户点击匹配结果中的技能卡片可快速激活
- [ ] 匹配结果为空时不展示推荐区
- [ ] MatchFeedbackWidget 在匹配结果旁展示，支持快速反馈
